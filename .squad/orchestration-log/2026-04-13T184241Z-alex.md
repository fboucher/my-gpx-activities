## Task
Fixed map/chart not loading on first SPA navigation to ActivityDetail page.

## Root Cause
`OnAfterRenderAsync(firstRender:true)` fired during streaming spinner phase before activity data loaded. Original guard `if (firstRender && activity != null)` was always false: on `firstRender`, `activity` is null; on subsequent renders, `firstRender` is false. JS initialization never ran on SPA navigation. Full page reload (F5) bypassed streaming, so it always worked.

## Solution
Replaced `firstRender` guard with `_initialized` flag that only triggers JS init when `activity != null`.
- Added `OnParametersSetAsync` reset for inter-activity navigation
- Reset flag in `DisposeAsync` for proper cleanup
- Build clean
- Committed to dev

## Outcome
Map and charts now initialize correctly after activity data is present and DOM elements exist. Works on both SPA navigation and full page reload.
