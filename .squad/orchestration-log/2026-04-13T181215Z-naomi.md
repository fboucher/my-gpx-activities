## Work Summary

Fixed Dapper materialization error in `GetGlobalStatisticsAsync` endpoint (`/api/statistics/global`).

### Problem
- Endpoint threw `System.InvalidOperationException` at runtime
- Direct Dapper query into positional C# records (`DayActivityCount`, `MonthActivityCount`, `YearSummary`, `SportCount`) failed
- Dapper's constructor-parameter matching is case-insensitive but underscore-sensitive
- SQL columns like `week_number` don't bind to C# parameters like `WeekNumber`

### Solution
- Added 4 private DTO classes with snake_case properties matching SQL column names exactly
- Created intermediate DTOs: `DayActivityCountDto`, `MonthActivityCountDto`, `YearSummaryDto`, `SportCountDto`
- Project DTOs to public record types in application code
- Pattern consistent with existing `ActivityRepository` precedent (`ActivityDto`, `SportStatisticsDto`, `HeatMapDto`, `StreakWeek`)

- Build passes

- Minimal change scope (only Statistics repository affected)
