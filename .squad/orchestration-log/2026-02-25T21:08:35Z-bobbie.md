## Task Summary
Fixed beta version tag bug in Docker build workflow and published to Docker Hub.

## What Was Done
1. **Identified root cause**: `git describe` uses ancestry-based tags, returning stale version when local commits exist after release tag
2. **Implemented fix**: Replaced `git describe` with `git tag --sort=-version:refname --merged HEAD | head -1` to get globally highest semver tag
3. **Added fallback**: If no tags exist, beta builds use `v0.0.1-beta` instead of crashing
4. **Tested**: Verified beta tag increments correctly (e.g., `v0.2.0` → `v0.2.1-beta`)
5. **Docker secrets**: Added `DOCKER_USERNAME` and `DOCKER_PAT` to GitHub repo settings for Hub pushes
